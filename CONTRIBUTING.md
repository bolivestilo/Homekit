# Contributing to Homekit

    Thank you for your interest in contributing! Homekit is an open project and all contributions are welcome.

    ## Getting started

    1. **Fork** the repo and clone your fork
    2. **Install dependencies**: `npm install`
    3. **Create a branch**: `git checkout -b feat/your-feature-name`
    4. **Make your changes** and commit with a clear message
    5. **Push** to your fork and open a pull request

    ## Development setup

    ```bash
    git clone https://github.com/bolivestilo/Homekit.git
    cd Homekit
    npm install
    npm run build
    ```

    Run in watch mode:

    ```bash
    npm run dev
    ```

    ## Commit style

    Use [Conventional Commits](https://www.conventionalcommits.org/):

    ```
    feat: add homekit status command
    fix: resolve scene export encoding issue
    docs: update MCP setup guide
    chore: bump dependencies
    ```

    ## Pull request checklist

    - [ ] Code follows the existing style
    - [ ] Tests added or updated where applicable
    - [ ] Documentation updated if needed
    - [ ] `CHANGELOG.md` updated under `[Unreleased]`
    - [ ] All CI checks pass

    ## Good first issues

    Check the [good first issue](https://github.com/bolivestilo/Homekit/issues?q=is%3Aopen+is%3Aissue+label%3A%22good+first+issue%22) label for beginner-friendly tasks.

    ## Questions?

    Open a [discussion](https://github.com/bolivestilo/Homekit/discussions) or ping the maintainer via a GitHub issue.
